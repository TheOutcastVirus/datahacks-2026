How We Got Here:
From La Jolla to Maine

This project didn't start in Maine. It started in La Jolla — with CalCOFI surface data, Argo heat content, and a neural network that kept losing to a one-line persistence baseline. Seven model iterations, two coasts, and one fundamental rethink later, we built something that actually works. This page is the honest account of that process: what we tried, what broke, what the data taught us, and why the final architecture looks nothing like the first one.

01

The La Jolla Problem

What We Started With

The original project targeted monthly sea-level anomaly (SLA) at La Jolla, CA (NOAA CO-OPS station 9410230, 1949–2021) with a 1–3 month forecasting horizon. The driving question: can ocean observations — specifically the Scripps CalCOFI hydrographic archive and the Argo global float network — predict upcoming sea-level departures before they happen?

The baseline was deliberately brutal: persistence, meaning "next month's SLA equals this month's SLA." If a model can't beat that in repeated out-of-sample testing, it has no practical value for coastal planning. Everything we built was judged against it.

The Data

Three sources formed the initial stack:

  • NOAA CO-OPS 9410230 — 1,217 monthly mean sea-level values (1949–2021), with seasonality removed to form the SLA target
  • CalCOFI bottle + cast archive — 35,644 hydrographic profiles from the California Cooperative Oceanic Fisheries Investigations, aggregated to monthly surface temperature and salinity within 1.5° of La Jolla
  • Argo EasyOneArgoTSLite (Scripps) — 2.7 million profiles streamed from a 1.5 GB tarball, filtered to a NE Pacific box (25–40°N, 140–118°W) and integrated 0–700 dbar to a monthly heat-content anomaly series

The early EDA finding set the tone for everything that followed: CalCOFI surface temperature correlated with SLA at r = 0.61 — but contemporaneously, not as a lead. Temperature and sea level respond to the same offshore state at the same time. That's information, but not a forecasting advantage.

02

Seven Model Iterations

Every iteration had a clear hypothesis. Most were wrong in an instructive way. The path from iteration 1 to iteration 6 is the story of a model that learned its place — and a team that learned what the data can and can't support.

Iteration 1 — Dense NN on raw anomalies

3-month input window (temp + SLA), 2 hidden layers, 64 units, ~60k parameters. Predictions oscillated violently between 3-month segments — far noisier than the actual. Monthly SLA anomalies have very little month-to-month memory, so the 3-month window was too short for any ENSO-scale signal. Each prediction head was independent; segments didn't line up.

Fails badly — adds noise
Iteration 2 — Smoothed target + residual head

Key improvements: 3-month centered rolling mean of SLA as target, 12-month input window, predicting the residual from persistence rather than an absolute anomaly. Smaller model (1×16) with weight decay. The NN did catch the 2015–16 El Niño peak qualitatively — but overall RMSE was 0.053 m vs. persistence's 0.034 m. Segment discontinuities remained.

NN 0.053 m / Persistence 0.034 m — loses by 56%
Iteration 3 — Add Argo 0–700 dbar heat content (inner join)

Integrated the Argo 1.5 GB tarball in a single streaming pass. Inner-joined onto the feature table — which silently collapsed training data from 700 months to 60, because CalCOFI reaches 1949 but Argo only starts in 2002. 60 training samples, 72 input features, ~1,200 model parameters. Training loss: 0.0001 (pure memorisation). Validation bias: −0.15 m below actual.

NN 0.123 m / Persistence 0.015 m — 8× worse (overfit)
Iteration 4 — Left-join Argo + honest date split

Left-joined Argo; filled pre-2002 values with 0 plus a binary argo_available mask. Hard date split at 2010, putting the 2015–16 El Niño in validation. Recovered training size — but created a severe distribution shift: argo_available = 0 for 228/318 training months, and 1 for every validation month. After z-scoring with training stats, validation Argo inputs were out-of-distribution.

NN 0.112 m / Persistence 0.034 m — distribution shift
Iteration 5 — Argo-era only, 3-in/1-out

Narrowed the problem: inner-join Argo with WINDOW=3, HORIZON=1, so every row has every feature. ~80 training windows (2002–2009), ~130 validation windows (2010–2020). Dramatically more stable than iterations 3–4 — smooth predictions, no explosions. But it completely missed the 2015–16 El Niño peak. The only ENSO-scale event in the Argo-era training window was the weak 2009–10 event. The 1982–83 and 1997–98 El Niños — the only real comparators for 2015–16 — were in the pre-Argo record the model never saw.

Stable but misses 2015 El Niño — insufficient training history
Iteration 6 — Full history, no Argo (the breakthrough)

Dropped Argo entirely and extended training back to 1950. Same window/horizon as iteration 5, only temp anomaly + rolling means + smoothed SLA. 327 training windows (1950–2009), 53 validation windows (2010–2021). Now includes 1982–83 and 1997–98 El Niños in training. The model tracked the 2015–16 peak closely and produced no pathological bias. 327 training windows vs. 60: data coverage was the biggest lever the whole time.

NN 0.0202 m / Persistence 0.0194 m — ties persistence ✓
Iteration 7 — Full history + CalCOFI basin-scale heat proxy

Built a CalCOFI proxy for 0–500 m heat content spanning 1949–2021 using a broad offshore SoCal box (29–36°N, 125–117°W) — the idea being to get subsurface information that spans the full record and includes the major ENSO events. Script written and checked in, but the per-profile Python integration loop hung before end-to-end completion. This was the state of the project when we recognized it was time to step back and rethink the overall strategy.

Not completed — vectorization fix needed

03

What the Data Taught Us

Six serious model configurations, one result that actually holds up. The lessons were hard-won and shaped every decision in the second model.

Lesson 01

Monthly SLA is near the predictability limit

La Jolla monthly sea-level anomaly at 1–3 month horizons is dominated by weather residual and tidal structure with very little month-to-month memory. No configuration clearly beat persistence. The best outcome was a tie. That's the honest story — and it shaped our choice to switch to a longer-horizon reconstruction problem where predictability exists.

Lesson 02

Data coverage beats model complexity

The single biggest jump — from "misses the 2015 El Niño" to "tracks it and ties persistence" — came from dropping Argo and extending the training window from 80 to 327 samples. Not a better architecture. Not more features. Just having the 1982–83 and 1997–98 events in training. Coverage first, complexity second.

Lesson 03

Argo doesn't lead SLA at 1–3 months here

The lagged-correlation plot was definitive: Argo 0–700 dbar heat content in the NE Pacific peaks at lag 0 against La Jolla SLA. Both series respond to the same offshore state simultaneously. Argo is genuinely valuable for longer horizons and broader spatial integrations — but it was the wrong tool for a 1-3 month local forecast.

Lesson 04

Smooth the target, predict the residual

Raw monthly SLA is too noisy for a direct multi-step forecast head. Predicting smoothed[t+k] − smoothed[t_last] — the change on top of persistence — removed the mean-level offset and focused learning on the departure from "stay put." This carried forward into how we handle deseasonalization in the Maine model.

Lesson 05

Distribution shift is silent and catastrophic

Left-joining Argo with a binary availability mask seemed clever — it preserved training size while flagging missingness. In practice, z-scoring with training stats sent validation Argo inputs completely out of distribution (availability: 0 for 72% of training, 1 for 100% of validation). The RMSE went from 0.034 m to 0.112 m. Hard temporal splits and explicit regime labels are non-negotiable.

Lesson 06

Salinity adds almost nothing locally

Surface salinity correlation with SLA at La Jolla: r = −0.132. After climatology removal, temperature and salinity anomalies were essentially uncorrelated (r = −0.002) and salinity made every feature set worse at every CalCOFI radius tested. In the Maine model we use Argo-derived density — which encodes both — as a single physically grounded predictor rather than separate T and S columns.

04

The Pivot

The realization that changed everything

After iteration 6, we had a model that tied persistence on a single California station at a 1-month horizon. That's a respectable result — but it isn't a planning tool. No city engineer uses "next month's sea level equals this month's sea level" to design a seawall. The forecasting problem was fundamentally under-constrained: too noisy, too local, too short-horizon.

The real planning need isn't "what happens next month." It's "what does sea level look like here in 2040, and how much year-to-year variability should we expect?" That's a reconstruction and projection problem — and it has a much richer observational foundation to build on.

What changed — Model v1 vs. Model v2

Model v1 — La Jolla (what broke)

  • Target: monthly SLA at a single California station, 1–3 month forecast horizon
  • Approach: neural network with multiple hidden layers
  • Argo used as a forecasting predictor (wrong use — it's lag-0, not lead)
  • Best result: tied persistence (0.0202 m vs 0.0194 m)
  • No multi-station comparison, no uncertainty quantification
  • Training window constrained by CalCOFI-Argo overlap, not station record length
  • No regime labeling — model was applied uniformly regardless of predictor availability

Model v2 — Maine (what works)

  • Target: historical reconstruction + century projection for Portland and Bar Harbor
  • Approach: ridge-regularized OLS — interpretable, validated, sample-efficient
  • Argo used as a calibration constraint and regime gate (correct use — density encodes thermohaline state, not future sea level)
  • Best result: R² = 0.861 on the 40-month calibration window; beats trend-only on rolling holdouts
  • Three stations, explicit cross-validation, 80% prediction intervals
  • Training exploits the full Portland record (113 years) via NAO
  • Three-regime output labeling: constrained / continuation / extrapolation

Why Maine?

The Gulf of Maine is one of the fastest-warming ocean regions on Earth and sits in the Northeast US sea-level "hotspot" identified by the 2022 federal technical report. Portland's tide gauge has run since 1912 — the longest continuous record in New England — giving us 113 years of annual means for the projection track. The NAO is a strong atmospheric driver of Gulf of Maine sea level and has a record extending back to 1950, providing the long predictor family that Argo and CalCOFI could not. And unlike California's single-station forecasting problem, the Maine coast has real municipal planning needs: Rockland's harbor, Portland's waterfront, and Bar Harbor's ferry infrastructure all face near-term sea-level decisions where a locally-calibrated model adds genuine value beyond the national projections.

Technical Stack

The full pipeline from raw data ingestion through model training and projection is implemented in Python, reproducible from source, and self-contained within the model02/ directory.

Data Ingestion

  • coops_tide_gauges.py — NOAA CO-OPS API
  • copernicusmarine_sla.py — Copernicus Marine DUACS
  • grace_mass_grids.py — NASA PO.DAAC GRACE/GRACE-FO
  • greenland_mass_timeseries.py — PO.DAAC Greenland mascon
  • argo_cube.py — EasyOneArgo NW Atlantic shelf cube
  • nao_index.py — NOAA CPC monthly NAO
  • land_motion.py — Portland GIA + VLM metadata

Preprocessing & Features

  • normalize_active_data.py — align to shared monthly timeline, deseasonalize, GIA-adjust
  • analyze_salinity.py — Argo shelf density derivation
  • reduced_budget_analysis.py — observational budget closure
  • annual_projection_data.py — Portland annual training table + NAO joins

Modeling

  • hindcast_model.py — ridge OLS ladder, 5-fold forward CV, final model selection
  • predict_monthly.py — regime-labeled monthly reconstruction export
  • annual_projection_model.py — annual model ladder, rolling 5-yr / 10-yr holdouts
  • project_annual.py — 4,000-path AR(1) bootstrap projection

Core Libraries

  • numpy, pandas — data wrangling
  • xarray, netCDF4 — NetCDF I/O
  • matplotlib — all diagnostic plots
  • copernicusmarine — altimetry download
  • podaac-data-subscriber (Python 3.12 runtime) — PO.DAAC download

Dashboard

  • Chart.js 4.4 — all interactive charts
  • PapaParse 5.4 — CSV loading
  • Vanilla HTML / CSS / JS — no build step
  • Serve with python -m http.server 8000

Validation Design

  • Monthly: 5-fold forward time-series CV (no future leakage)
  • Annual: rolling-origin 5-yr and 10-yr holdouts + contiguous 10-yr tail
  • Baselines: persistence, trend-only, seasonal climatology
  • Metrics: RMSE, MAE, bias, trend error, 80% interval coverage

Why ridge OLS, not a neural network?

The La Jolla iterations showed what happens when model complexity outruns sample size. The validated Argo-overlap window for Maine is 40 months — far too small for any deep architecture. Ridge regression is optimal here: it handles near-collinear predictors (SLA, mass, density all correlate), produces interpretable coefficients, cross-validates in seconds, and generalizes on small windows where a NN would memorize. The R² = 0.861 result would almost certainly degrade to negative territory if we swapped in a two-layer NN without an order of magnitude more training data.
